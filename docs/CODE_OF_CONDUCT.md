# Code of conduct

## Our standard

Participation in nv0ken is expected to be respectful, technically honest, and
constructive. We welcome contributors with different levels of systems
experience and expect discussion to focus on the work rather than the person.

Examples of expected behavior include:

- Giving actionable technical feedback and accepting correction gracefully.
- Respecting differing viewpoints and explaining trade-offs with evidence.
- Reporting security and safety issues through the private process in
  [SECURITY.md](SECURITY.md).
- Assuming good intent while still challenging unsupported claims.

Unacceptable behavior includes harassment, discriminatory language, threats,
personal attacks, deliberate disruption, publishing private information, or
misrepresenting test results and third-party work as original.

## Enforcement

Project maintainers may remove, edit, or reject contributions and may restrict
participation when behavior conflicts with this standard. Reports should include
the relevant context and be sent privately to the repository maintainers via
the GitHub security contact or a private repository contact channel. Reports
will be handled as confidentially as practical.
